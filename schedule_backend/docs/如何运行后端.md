# Schedule App 后端 - 运行指南

## 一、首次使用（环境准备）

### 1. 安装 Python 依赖

```powershell
cd schedule_backend
uv sync
```

`uv sync` 会创建 `.venv` 并安装依赖。**用 `uv run` 时自动使用该虚拟环境**，无需手动 `activate`。

若 `uv sync` 超时，可编辑 `pyproject.toml` 切换镜像源，或改用 pip：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
```

### 2. 配置环境变量

```powershell
cp .env.example .env
# 用记事本或 VS Code 编辑 .env，填入：
# - MINIMAX_API_KEY（或 DEEPSEEK_API_KEY / OPENAI_API_KEY）
# - DATABASE_URL（PostgreSQL 连接串）
```

### 3. 启动数据库

**本地已有 PostgreSQL：**

```powershell
uv run python scripts/create_db.py   # 若数据库不存在则创建
uv run alembic upgrade head          # 执行迁移
uv run python scripts/seed.py        # 可选：写入测试数据
```

**本地无 PostgreSQL：**

```powershell
docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=postgres pgvector/pgvector
# 然后执行上述 create_db、alembic、seed
```

### 4. 构建后台监控（可选）

```powershell
cd admin-ui
npm install
npm run build
cd ..
```

## 二、日常启动

每次开发或调试时，只需：

```powershell
cd schedule_backend
uv run python run.py
```

**若要看用量监控后台**：需先启动后端，再 `cd admin-ui` → `npm run dev`，访问 http://localhost:5174/admin/

或：

```powershell
cd schedule_backend
uv run uvicorn app.main:app --reload --port 8000
```

服务启动后：

- API：http://localhost:8000
- 文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health
- 用量监控：http://localhost:8000/admin

## 三、常用命令速查

| 命令 | 说明 |
|------|------|
| `uv run python run.py` | 启动后端 |
| `uv run alembic upgrade head` | 执行数据库迁移 |
| `uv run python scripts/seed.py` | 写入种子数据 |
| `uv run python scripts/create_db.py` | 创建数据库（若不存在） |
| `cd admin-ui && npm run build` | 构建后台监控 |
| `cd admin-ui && npm run dev` | 开发模式后台（端口 5174） |

## 四、Flutter 对接

在 `schedule_app_flutter/lib/services/api_service.dart` 中设置 `baseUrl`：

- 模拟器：`http://10.0.2.2:8000/api`
- 真机：`http://你的电脑IP:8000/api`（如 `http://192.168.1.100:8000/api`）

## 五、常见问题

**Q：数据库连接失败？**  
检查 `.env` 中 `DATABASE_URL` 的用户名、密码、数据库名是否正确。

**Q：AI 调用失败？**  
检查 `.env` 中是否配置了 `MINIMAX_API_KEY` 或 `OPENAI_API_KEY`。

**Q：端口 8000 被占用？**  
在 `.env` 中设置 `PORT=8001` 或修改 `uv run uvicorn` 的 `--port` 参数。
