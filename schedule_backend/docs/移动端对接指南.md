# 移动端对接后端指南

## 一、前置条件

1. 后端已启动：`uv run python run.py`（运行在 8000 端口）
2. 电脑与手机/模拟器在同一局域网（真机时）

---

## 二、baseUrl 配置

**模拟器**：已自动配置，无需修改。
- Android 模拟器 → `10.0.2.2:8000`
- iOS 模拟器 → `127.0.0.1:8000`

**真机**：在 `main.dart` 的 `main()` 开头添加：

```dart
import 'services/api_service.dart';

void main() async {
  ApiService.overrideBaseUrl = 'http://192.168.1.100:8000/api';  // 换成你的电脑 IP
  WidgetsFlutterBinding.ensureInitialized();
  // ...
}
```

查看电脑 IP：`ipconfig`（Windows）或 `ifconfig`（Mac/Linux）。

---

## 三、如何验证效果

1. **启动后端**：`cd schedule_backend` → `uv run python run.py`
2. **启动 Flutter**：`cd schedule_app_flutter` → `flutter run`
3. **进入 AI 聊天**：在 App 中找到 AI 聊天入口（如日历页的聊天按钮、聊天页等）
4. **发送消息**：输入任意文字并发送，应收到 MiniMax 的真实 AI 回复

---

## 四、验证清单

- [ ] 后端终端无报错，显示 `Uvicorn running on http://0.0.0.0:8000`
- [ ] 浏览器访问 http://localhost:8000/docs 可打开 API 文档
- [ ] Flutter 已切换为 `RemoteAIChatRepository`（见 `main.dart`）
- [ ] `api_service.dart` 的 `baseUrl` 与当前运行环境匹配

## 五、常见问题

**Q：App 显示「网络连接失败」？**  
- 检查 baseUrl 是否正确（模拟器用 10.0.2.2，真机用电脑 IP）
- 确认后端已启动
- 真机时确认手机和电脑在同一 WiFi

**Q：App 显示「请求失败：500」？**  
- 已连上后端，但后端处理失败。**查看后端终端**的完整报错（红色堆栈）
- 常见原因：MiniMax API Key 无效/过期、余额不足、数据库连接失败
- 修复后 Flutter 会显示后端返回的具体错误信息（如 `MiniMax API 错误: 401`）

**Q：管理界面显示「加载失败: HTTP 500」？**  
- 用量接口依赖数据库。确认：1) PostgreSQL 已启动 2) 已执行 `uv run alembic upgrade head` 3) `.env` 中 `DATABASE_URL` 正确

**Q：用量统计里看不到自己？**  
- Flutter 会在首次发 AI 消息时把本地用户 ID 设为 `user_001`（与后端 seed 一致），并在请求里带 `userId` / `X-User-Id`  
- 完成登录后会改为 `user_{手机号}`。管理后台「按用户排行」里对应这两类 ID
