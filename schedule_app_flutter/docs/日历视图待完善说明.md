## 日历视图（日 / 周 / 月）与 iOS 对齐计划

### 1. 当前 Flutter 实现现状（2026-02-03）

- **入口页面**
  - `MainContainerView` 使用 `LeafButtonGroup` 作为日 / 周 / 月三叶草按钮，当前只更新本地 `_calendarMode`，并通过 `SnackBar` 提示，**尚未真正驱动日历视图切换**。
  - `CalendarPage` 目前只实现了类似 iOS `DayCalendarView` 的「单日任务列表」，没有独立的周视图 / 月视图组件。

- **叶子按钮位置**
  - 已调整为：`top = statusBar + 60(导航栏) + 10(卡片顶部偏移) + 30(叶子距卡片顶部偏移)`，使第一个“月”按钮刚好位于日历大卡片顶部下方，和 iOS 对齐。

- **日视图顶部区域**
  - Flutter：`CalendarPage` 顶部显示的是「当前日期 + 左右切换箭头 + 简单天气文本」，日期格式为「MM月dd日 星期X」，参考 `DayCalendarView` 的 UI。
  - 还有一个日期选择对话框 `_pickDate()`，目前仅用于跳到任意日期，不区分视图模式。

### 2. 与 iOS 差异 / 待完成点

1. **周 / 月视图组件缺失（已补上基础版本）**
   - iOS:
     - `WeekCalendarView`：横向列出一周 7 天 + 纵向时间刻度，点击具体任务弹出详情。
     - `MonthCalendarView`：整月网格视图，可点击某天，底部用小圆点表示任务分布。
   - Flutter:
     - 新增 `widgets/week_calendar_view.dart`：
       - 显示一周 7 天的横向日期条，支持「上一周 / 下一周」切换；
       - 高亮今天和选中日期，选中某天时在下方显示这一天的任务列表；
       - 通过 `onDateSelected(DateTime)` 回调选中的日期（用于后续跳转到日视图）。
     - 新增 `widgets/month_calendar_view.dart`：
       - 显示当前月份的 6x7 日期网格，支持「上一月 / 下一月」切换；
       - 当天高亮显示；每日底部用最多 3 个彩色小圆点表示任务数量（颜色对应 `SpiritType`）；
       - 点击某天通过 `onDateSelected(DateTime)` 回调日期。
     - 目前这两个组件**尚未在 `CalendarPage` 中接入 `_calendarMode` 切换逻辑**，只是提供了独立可复用的周 / 月视图组件，方便后续在阶段 3 中挂载到 `CalendarPage`。

2. **三叶草按钮与日历模式联动**
   - iOS:
     - `CalendarViewController` 中的三叶草按钮点击后会调用 `selectCalendarMode(_:)`，进而切换 `currentCalendarView` 为日 / 周 / 月三种不同子视图，并在模式切换时把任务数组传入对应视图。
   - Flutter（现在的实现）:
     - `LeafButtonGroup` 通过回调把 `CalendarMode` 回传给 `MainContainerView`，`MainContainerView` 持有 `_calendarMode` 状态：
       - 作为三叶草按钮的 `initialMode`，保证按钮选中态与当前模式同步；
       - 传入 `CalendarPage(initialMode: _calendarMode)`，驱动内部显示「日 / 周 / 月」三种视图。
     - `CalendarPage`：
       - 新增参数 `initialMode: CalendarMode` 和内部状态 `_calendarMode`；
       - 根据 `_calendarMode` 切换内容区域：
         - `CalendarMode.day`：沿用原来的「日视图」布局（日期 + 天气 + 精灵筛选 + 列表）；
         - `CalendarMode.week`：使用 `WeekCalendarView(anchorDate: _selectedDate, tasks: allTasks, onDateSelected: _jumpToDay)`；
         - `CalendarMode.month`：使用 `MonthCalendarView(anchorDate: _selectedDate, tasks: allTasks, onDateSelected: _jumpToDay)`；
       - 提供 `onModeChanged` 回调，在 `_jumpToDay(DateTime date)` 中切回 `CalendarMode.day` 时通知外层：
         - `MainContainerView` 收到回调后同步更新 `_calendarMode`，三叶草高亮自动切回「日」。
   - **目前状态小结**:
     - 从「三叶草按钮」→ `_calendarMode` → `CalendarPage` 的模式状态已经打通；
     - 从「周 / 月视图中点击某一天」→ `CalendarPage._jumpToDay` → `onModeChanged` → `MainContainerView` → 三叶草按钮 的链路也已接上；
     - 后续若要调整 UI 细节（例如周 / 月模式下是否显示精灵筛选器、是否区分不同模式的头部样式），只需要在 `CalendarPage` 内部按 `_calendarMode` 分支微调即可，不需要再改动状态传递链路。

3. **点击周 / 月里的某一天跳转到日视图**
   - iOS:
     - `MonthCalendarView` 和 `WeekCalendarView` 都通过 `CalendarViewDelegate.didSelectDate(_:)` 回调日期。
     - `CalendarViewController.didSelectDate(_:)` 内部调用 `jumpToDay(date:)`：
       - 切换模式到 `.day`
       - 更新 `DayCalendarView` 的当前日期
   - Flutter（现状，已实现）:
     - `WeekCalendarView` / `MonthCalendarView` 构造函数都接收同样的回调：
       - `final ValueChanged<DateTime>? onDateSelected;`
     - 在周视图 / 月视图中，点击某一天时会调用 `onDateSelected(date)`。
     - `CalendarPage` 中实现了 `_jumpToDay(DateTime date)`，并将其传给周 / 月视图：
       - 在 `_jumpToDay` 内部：
         - 更新 `_selectedDate = date`；
         - 将 `_calendarMode` 切换为 `CalendarMode.day`；
         - 调用 `widget.onModeChanged?.call(CalendarMode.day)` 通知外层。
     - `MainContainerView` 通过 `onModeChanged` 回调同步更新自身的 `_calendarMode`，从而让左侧三叶草按钮在「从周 / 月点击某一天跳转回日视图」时，也自动高亮到「日」这个叶子按钮，实现与 iOS 相同的交互链路。

4. **真实的天气信息**
   - iOS:
     - `DayCalendarView` 中有天气标签（目前示例写死为 "☀️ 晴 18°C"）。
   - Flutter:
     - `CalendarPage` 同样使用固定文案 `"☀️ 晴 18°C"` 作为占位。
   - **后续可选优化**:
     - 接入天气 API，根据 `_selectedDate` 和用户所在城市展示真实天气。

  - 任务布局 / 小圆点指示
  - 点击交互（任务详情弹层、删除回调等）可以先做简化版。

**阶段 4（点击周 / 月日期跳转日视图）**

- [ ] `CalendarPage` 实现：
  - 更新 `_selectedDate`

  - 触发刷新，显示对应日期的日视图任务列表。

> 备注：以上计划是为了严格对齐 iOS 的 `CalendarViewController + Day/Week/MonthCalendarView` 行为，具体实现过程中可以根据 Flutter UI 效果做微调，但交互路径和数据流应保持一致，方便后续联调与维护。
