# 光合桌面观察 · 运行教程

桌面端工程在 monorepo 的 **`SchedulePCagent`** 目录下，**不要在仓库根目录 `ScheduleApp` 里执行 `npm run tauri:dev`**，否则会报错找不到 `package.json`（`ENOENT`）。

---

## 一、环境要求

| 依赖 | 说明 |
|------|------|
| **Node.js** | 建议 18+（与 `package.json` 中工具链一致即可） |
| **Rust** | `rustc` / `cargo`，用于编译 Tauri 与 `harness-core` |
| **Windows** | 需 **Visual Studio Build Tools（含 MSVC）** 或完整 VS，以使用 `x86_64-pc-windows-msvc` 目标 |

### Rust 装在 D 盘（可选）

若 Rust 通过 `RUSTUP_HOME` / `CARGO_HOME` 装在非默认路径（例如 `D:\Rust`），请保证：

- 新开终端后 `cargo -V` 可用；或  
- **直接使用本仓库脚本**：`npm run tauri:dev` 会调用 `scripts/run-with-rust-env.mjs`，为本次进程把 `D:\Rust\cargo\bin` 加入 `PATH`（与 `SchedulePCagent/README.md` 一致）。

---

## 二、第一次运行（推荐步骤）

在 **PowerShell** 或 **cmd** 中执行（注意先进入子目录）：

```powershell
cd D:\Users\zyx\Desktop\ScheduleApp\SchedulePCagent
npm install
npm run tauri:dev
```

> **PowerShell 老版本** 不支持 `&&` 链式命令，请分行执行，或写成：  
> `cd D:\...\SchedulePCagent; npm install; npm run tauri:dev`

成功时你会看到：

- 终端里 **Vite** 提示 `http://localhost:1420/`
- 随后 **cargo** 编译并启动 `schedule-pc-agent.exe`
- 桌面上弹出 **主窗口「光合桌面观察」** 和 **气泡球窗口「光合日历」**

主界面功能：开始/停止记录、刷新、AI 分析、显示气泡球等，详见 [`README.md`](../README.md)。

### 方舟 AI（可选）

若要用「AI 分析今日」：

1. 复制 `.env.example` 为 **`.env`**
2. 填写 `ARK_API_KEY=...`（勿提交 `.env`）

## 三、常见错误与处理

### 1. `Could not read package.json` / `ENOENT`

**原因**：当前目录不是 `SchedulePCagent`。

**处理**：

```powershell
cd D:\Users\zyx\Desktop\ScheduleApp\SchedulePCagent
```

再执行 `npm run tauri:dev`。

---

### 2. `cargo` 不是内部或外部命令

**原因**：当前终端未加载 Rust 的 `PATH`。

**处理**：新开终端并配置好 `Path`；或仍在本目录执行 **`npm run tauri:dev`**（由 `run-with-rust-env.mjs` 注入 D 盘 cargo 路径，若你机器上路径不同，请改脚本或自行配置环境变量）。

---

### 3. 编译时 `failed to remove ... schedule-pc-agent.exe` / 拒绝访问

**原因**：上一版 exe 仍在运行，文件被占用。

**处理**：关闭「光合桌面观察」和气泡窗口；任务管理器结束 `schedule-pc-agent`，或在 PowerShell：

```powershell
Stop-Process -Name "schedule-pc-agent" -Force -ErrorAction SilentlyContinue
```

再重新 `npm run tauri:dev`。

---

### 4. 浏览器打开 `localhost:1420` 白屏或提示不是 Tauri

**原因**：调试壳依赖 **Tauri 内嵌 WebView**，单独用浏览器打开 Vite 地址无法调用 `invoke`。

**处理**：使用 **`npm run tauri:dev` 弹出的桌面窗口**，不要只靠浏览器标签页。

更多说明见 [`数据与运行说明.md`](./数据与运行说明.md)。

## 四、其他常用命令

| 命令 | 作用 |
|------|------|
| `npm run dev` | 仅启动 Vite（一般不单独用于完整桌面功能） |
| `npm run build` | 前端生产构建 |
| `npm run tauri:build` | 打桌面安装包/可执行文件（需完整 Rust 环境） |
| `npm run export:timeline-today` | 导出今日时间线 JSON（见 `data/export/`） |

## 五、与仓库其它部分的关系

- **Flutter / 后端 / 管理台**：在仓库根目录 [`README.md`](../../README.md) 的「快速启动」中说明；路径分别为 `schedule_app_flutter`、`schedule_backend`、`admin-ui`。
