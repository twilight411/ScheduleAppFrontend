# 光合桌面观察 · 数据落盘与界面（白屏）说明

本文说明：**`.db` / `.wal` / `.shm` 是什么**、**数据存在哪（为何曾默认在 C、现在怎么放 D/仓库）**、**每条记录里的 JSON 在哪**、**怎么导出给 AI**、以及 **WebView / 停录与丢数据**。

---

## 0. `.db`、`.db-wal`、`.db-shm` 是什么？

这是 **SQLite** 在磁盘上的常见三件套（你开了 **WAL 模式** 时）：

| 文件 | 作用（直观理解） |
|------|------------------|
| **`harness.db`** | **主库文件**：表结构、索引、以及大部分已提交的数据页。 |
| **`harness.db-wal`** | **Write-Ahead Log**：新写入先追加在这里，再择机合并回主库。运行中常有数据「还在 WAL 里」的情况。 |
| **`harness.db-shm`** | **共享内存索引**：配合 WAL 用的辅助文件，体量通常很小。 |

**不要**在程序正在跑的时候只拷贝一个孤零零的 `.db` 就认为备份完整——**应同时拷贝 `.db` + `.wal` + `.shm`**，或先**正常退出**应用再拷贝（让 SQLite 有机会 checkpoint）。否则用 DB 浏览器打开可能看到**旧快照**或报错。

## 2. 数据存在哪里？为什么以前像在「C 盘」？

### 2.1 以前为什么默认在 C？

很多桌面程序默认把「用户数据」放在 **`%LOCALAPPDATA%`**（Windows 上通常是 `C:\Users\<用户名>\AppData\Local\`）。这不是我们「故意」写死 C，而是操作系统约定：**用户目录往往在系统盘**。早期示例代码也用过 `ScheduleHarness` 子目录。

### 2.2 现在默认放哪？

优先使用 **本仓库下的 `SchedulePCagent/data/harness.db`**（你的项目在 D 盘时，数据就在 D 盘）。

实现上的优先级是：

1. **`HARNESS_DB_PATH`**：直接指定 `.db` 的完整路径（最高优先级）。
2. **`GUANGHE_DATA_DIR`**：目录，程序使用其中的 `harness.db`。  
   **`npm run tauri:dev` / `tauri:build`** 会通过脚本自动设为仓库里的 `data/`。
3. 当前工作目录下存在 **`package.json`**（即在 `SchedulePCagent` 里跑 `cargo run -p harness-cli`）→ 使用 **`./data/harness.db`**。
4. 以上都不满足时的兜底：`%LOCALAPPDATA%\GuangheDesktopObserve\harness.db`（仍可能在 C，仅作兼容）。

详见仓库内 [`data/README.md`](../data/README.md)。

### 2.3 从旧路径迁到 `data/`

若你以前在 `%LOCALAPPDATA%\ScheduleHarness\` 已有库，可**退出程序后**把整个 `harness.db*` 复制到 `SchedulePCagent/data/`，再启动应用。

### 2.4 关掉窗口 / 停录 → 数据丢吗？

**不会**因为停录或关窗口就删掉已写入磁盘的内容；详情见下文第 6 节。

## 3. 「不是 JSON 吗？」——JSON 在哪？

**仍然是 JSON，只是包在 SQLite 里。**

表 `desktop_events` 里有一列 **`payload`**，存的就是 **一条事件的 JSON 字符串**，例如：

- `type: "app_switch"` 时带 `app`、`title`、`at`
- `type: "clipboard"` 时带 `text_preview`、`char_len`、`truncated`、`at`

所以：**磁盘上是 `.db` 容器；逻辑上每条事件仍是 JSON。** 很多 AI/聊天框不能直接上传 `.db`，需要 **导出成 `.json` 文本**（见下一节）。

---

## 4. 怎么把数据发给 AI？

1. **推荐（本仓库脚本）**  
   在 `SchedulePCagent` 下执行：
   ```bash
   npm run export:timeline-today
   ```  
   会在 **`data/export/timeline-today-YYYY-MM-DD.json`** 生成**今日**事件的 JSON 数组，可直接打开复制，或把文件内容发给 AI。

2. **命令行**（需已配置 `cargo` 且在仓库根目录，或已设 `GUANGHE_DATA_DIR`）  
   ```bash
   cargo run -p harness-cli -- timeline today --json
   ```

3. **用 DB 浏览器**（如 DB Browser for SQLite）打开 `data/harness.db`，查看 `payload` 列并自行复制 JSON。

## 5. 库表与事件字段（对照「采集理念是否合理」）

| 列 | 含义 |
|----|------|
| `id` | 自增主键 |
| `recorded_at` | 事件时间（RFC3339，UTC 字符串） |
| `payload` | **整段 JSON**，一种 `DesktopEvent` |

### 5.1 `app_switch`

- `app`、`title`、`at`

### 5.2 `clipboard`

- `text_preview`、`char_len`、`truncated`、`at`  
- **隐私**：剪贴板可能含敏感信息；后续可改为只存哈希/长度或默认关闭。

## 6. WebView、停录、白屏（简述）

- **WebView**：画 UI、跑 React、通过 `invoke` 调 Rust。  
- **停录**：只停 Rust 里的轮询线程，**不删库**。  
- **白屏**：多半是页面/资源未加载或前端报错，与「数据是否在库里」无必然联系。  

详见下文原「白屏」表（第 7 节）。

---

## 7. 白屏常见原因与排查

| 现象 | 可能原因 |
|------|----------|
| 整窗纯白 | Vite 未在 `localhost:1420` 启动。 |
| 整窗纯白 | 打包后资源路径错误（已设 Vite `base: './'`，需先 `npm run build` 再 `tauri build`）。 |
| 浏览器能开、Tauri 不行 | 浏览器里没有 Rust 后端；应用内需 `isTauri()`。 |

**建议**：开发使用 **`npm run tauri:dev`**。
